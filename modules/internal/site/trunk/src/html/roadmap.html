Roadmap

Quokka consists of many modules. The core itself consists of over 10 modules and each extension, including plugins, archetypes and dependency sets are separate modules and have their own lifecycle.

All of this is necessary, but makes tracking releases more difficult. The approach quokka takes is to group together a set of module milestones into consolidated milestones (initially monthly). Consolidated milestones will include a new quokka distribution of the same name and will synchronise all extensions to work with the new core distribution.

All work on quokka is assigned to an issue within the issue tracker, including the milestone assinged to it.
